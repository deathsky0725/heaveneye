# t_9177d109 — plan.md/result.md auto-update hook

## What was done

Designed and implemented a post-task hook that appends structured entries to `result.md` when kanban events fire.

## Design decision

**Option B** chosen (integrate into existing kanban watcher `server/watchers/kanban.ts`) rather than a separate polling script or Hermes webhook — reuses existing board discovery + event handling infrastructure, no extra processes.

## Files changed

### NEW: `server/watchers/resultMdUpdater.ts`
- `resolveBoardResultPath(boardSlug)` — maps board slug → result.md absolute path
  - `heaveneye-ui` → `~/Documents/Agentic-OS/Projects/heaveneye/result.md`
  - `ytdsl-ep001` → `~/Documents/Agentic-OS/Projects/ytdsl-ep001/result.md`
  - Other slugs → heuristic `Projects/{slug}/result.md`
- `appendResultMdEntry(ResultMdEntry)` — atomic append (temp file + rename)
- Entry format: `## [{ts}] {agent} {event} → {title} [{tid}]` with status/task/board fields and optional reason

### MODIFY: `server/watchers/kanban.ts`
Wired hook into 4 event handlers:
- `completed` → appends to result.md
- `blocked` → appends with reason extracted from payload or latest agent comment
- `unblocked` → appends
- `crashed/gave_up/timed_out` → appends with event kind

### MODIFY: `result.md` (open items)
Updated item #2 to mark as shipped with reference.

## Entry format (example)
```
## [2026-05-19T12:00:00Z] yefan completed → Design + implement plan.md/result.md auto-update hook [t_9177d109]

  - Status: completed
  - Task: Design + implement plan.md/result.md auto-update hook (t_9177d109)
  - Board: heaveneye-ui
```

## What does NOT get written
- claimed / spawned / heartbeat — too noisy
- plan.md is intentionally NOT auto-updated (it's manual intent/roadmap)

## Self-review
- Board→path mapping is explicit for known boards, heuristic for unknown — correct fallback behavior
- Atomic write (temp + rename) avoids corruption on concurrent writes
- Skips silently if board has no resolvable result.md path — no noisy failures
- Front-matter detection skips code fences (`prevChar !== '\``) — handles markdown tables/blocks correctly
